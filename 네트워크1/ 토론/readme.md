# 토론
---
> 주제<br>
```
-가장 좋은 오류 제어 방법은?
```
> 제출<br>
```
먼저 네트워크 상에서 오류가 생기는 경우는 크게 두가지로 나눌 수 있습니다. 데이터 변형 오류와 데이터 분실오류가 있습니다.
데이터 변형오류는 데이터가 때져서 도착하는 것을 말하며, 데이터 분실오류는 데이터가 아예 도착하지 못하는 것을 말합니다.
이러한 오류가 발생하면 네트워크 상태가 불안정해지기 때문에, 오류 제어 기능은 통신 프로토콜에 있어 가장 기본적인 기능이라고 할 수 있습니다.
이러한 오류를 제어하는 방법에는 프레임 릴레이와 셀 릴레이라는 두가지 방법이 있는데,
이 두가지 오류제어 방법은 패킷 교환 방식이 고안된 시점에 디지털 통신을 할 때 전송 오류가 많이 발생하여 생겨나게 되었다는 공통점이 있으며,
- 프레임 릴레이 : 오류제어 기능을 단순화하는 작업으로 데이터 전송의 효율을 크게 향상시, CRC를 통한 오류감지와 상위 계층에서  재전송 처리하기에 주로 비용 효율적인 데이터 전송에 사용이 됩니다
- 셀 릴레이 : 오류제어에 대한 오버헤드를 최소화하는 방식으로 사용, 네트워크 내부에서 오류를 처리하며 손상된 정보를 자동으로 처리한 후 상위로 올려보네는 방법을 사용합니다.
즉, 두가지가 사용되는 상황은 각각 아래와 같이 구분이 됩니다.
프레임 릴레이는 저비용과 고속 전송이 필요한 상황인 기업의 사설 네트워크 구축, ISP의 원거리 통신, 음성과 데이터를 동시에 전송하는 경우에 적합합니다.
셀 릴레이의 경우는 고속 전송과 신뢰성있는 전송이 필요한 경우에 사용되며 이들의 예로는 금융기관의 데이터 센터들간의 데이터 처리(ATM), 영상 회의나 실시간 멀티미디어 통신(지연이 최소화 높은 품질 연결이 필요), 대용량 트래픽처리가 필요할떄 사용이 용이합니다.
위에서 보기와 같이 두가지 방식 모두 각자의 장점과 단점이 분명히 있어 어떤 오류 제어 방법이 휠씬 좋다고 생각되기 보다 자신의 네트워크 상황에 맞는 오류 제어 기능을 사용하는 태도가 필요하다고 생각됩니다.
```
