# 토론
---
> 주제<br>
```
-가장 좋은 오류 제어 방법은?
```
> 참고 자료<br>
```
- 네트워크 상에서 오류가 생기는 경우는 크게 두가지로 구분할 수 있다. 데이터 변형 오류와 데이터 분실오류가 바로 그것이다
- 데이터 변형오류는 데이터가 때져서 도착하는 것을 말하며, 데이터 분실오류는 데이터가 아예 도착하지 못하는 것을 말한다
- 이러한 오류가 생기면 원할한 네트워크 상태를 유지할 수 없기 때문에 오류 제어 기능은 통신 프로토콜에 있어 가장 기본적인 기능이라고 할 수 있다.
- 이러한 오류를 제어하는 방법에는 프레임 릴레이와 셀 릴레이라는 두가지 방법이 있다.
- 이 두가지 오류제어 방법은 패킷 교환 방식이 고안된ㄴ 시점에 디지털 통신을 할 때 전송 오류가 많이 발생하여 생겨나게 되었다.
- 프레임 릴레이 : 오류제어 기능을 단순화하는 작업으로 데이텅 전송의 효율을 크게 높임
- 엘 렐에이 : 오류제어에 대한 오버 헤드를 최소화하는 방식으로 사용
- 두가지 방식 모두 각자의 장점이 있어 어떤한 오류 제어 방법이 휠씬 좋다고 생각되기 보다 자신의 네트워크 상황에 맞는 오류 제어 기능을 사용하는 태도가 필요하다고 생각된다
==> 상황을 구분해서 나에게 적용된다면도 생각해서 서술해 보자
```
