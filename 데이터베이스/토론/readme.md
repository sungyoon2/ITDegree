# 토론
---
> 주제<br>
```
- 데이터베이스는 3단계 스키마 구성으로 이루어져 있다. 각 스키마 구성으로 이루어져 있다. 각 스키마 사이의 대응관계에서 발생하는 데이터 독립성은 반드시 지켜져야 하는 원칙은 아니며, 상황에 따라 지키거나 지켜도 되지 않는 자유로운 개념을 함께 설명하고 찬반에 대한 근거를 제시
```
> 제출<br>
```
먼저, 저는 데이터 독립성이 반드시 지켜져야한다는 입장입니다.
각각의 개념부터 언급하자면 데이터베이스의 3단계 스키마란 외부 스키마(사용자 관점), 개념 스키마(전체 구조), 내부 스키마(물리적 저장)를 말합니다.
또한 3단계 스키마간의 데이터 독립성은 논리적 독립성과 물리적 독립성을 포함합니다.
논리적 독립성은 개념 스키마 변경이 외부 스키마에 영향을 주지 않는 것을 의미하고, 물리적 독립성은 내부 스키마 변경이 개념 스키마에 영향을 미치지 않는 것을 뜻합니다.
위의 두 부분을 포함한 데이터 독립성은 시스템 유지보수성과 확장성을 높이는 핵심원칙이기 때문에 저는 데이터 독립성이 반드시 지켜져야한다는 입장입니다.
독립성이 없다면 물리적 저장 구조의 변경(내부 스키마의 변경)이 사용자 인터페이스(외부 스키마의 변경)나 응용 프로그램(개념 스카마의 변경)에 영향을 미치게 되어, 관리 비용과 오류 가능성이 크게 증가 합니다.
물론 특정 상황에서는 독립성을 완화할 수도 있지만, 이는 시스템의 효율성을 잠시 높일 수 있는 단기적인 편의일 뿐 장기적으로 시스템의 안정성과 유연성을 해칠수 있다고 생각하며, 이는 작은 잠시뿐인 이득을 얻기위해 큰것을 버리는 것으로 생각이 됩니다.
따라서 데이터 독립성은 데이터베이스 설계의 기본 원칙으로 지켜져야 한다고 생각합니다.
```
