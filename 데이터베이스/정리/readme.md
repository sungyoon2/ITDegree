# 데이터베이스 정리

- 현실세계에서 단순히 관찰하거나 측정해 수집한 사실이나 값

> 데이터 베이스
- 정보시스템에서 데이터의 저장과 제공에 관련된 역할
---
> 정보<br>
- 데이터를 처리하여 여러가지의 의미를 함축적으로 표현이 가능한 형태
- 데이터의 가공으로 산출된 정보는 또다른 정보산출을 위한 data로 사용가능
---
> 의사 결정 지원시스템<br>
- DSS
- 광의의 정보시스템
---
> 데이터베이스의 성질<br>
- 통합데이터
- 저장데이터
- 공유데이터
- 운영데이터 -> 조직의 주요기능 수행을 위해 꼭 필요함

> 데이터베이스의 특징<br>
- 실시간 접근
- 계속적인 변환(Update)
- 동시 공유
- 내용기반 참조(값으로 찾을 수 있음)

- 물리적 구조

```
저장 관점에서 본 데이터의 물리적 배치
```
- 논리적 구조
```
데이터의 개념적 구조
```
---
> 데이터베이스 관리 시스템<br>
- 데이터의 정의, 조작, 제어 기능을 제공
---
> 데이터 베이스의 단점<br>
- 높은 비용
- 백업 및 회복 방법이 복잡
- 중앙 집중 관리로 인한 취약점 존재
---
> 파일 시스템<br>
- 응용프로그램마다 필요한 데이터를 별도로 관리
- 데이터를 파일로 관리하기 위해 파일을 생성, 삭제, 수정, 검색기능을 제공하는 소프트웨어
---
> 파일 시스템의 문제점<br>
- 중복성
- 종속성
---
> DBMS 기능<br>
-정의
```
데이터베이스 구조를 정의, 수정
```
-조작
```
수정, 삭제, 검색등의 기능
```
-제어
```
데이터의 정확성과 안전성 유지
```
---
> 데이터 베이스 시스템과 데이터 베이스 관리 시스템<br>
```
1. 데이터 베이스 시스템
- 데이터 베이스에 데이터를 저샂ㅇ하고, 이를 관리하여 조직에 필요한 정보를 생성해주는 시스템
- 데이터 베이스 관리시스템보다 더 큰 의미
2. 데이터 베이스 관리 시스템
- 조직에 필요한 데이터를 데이터 베이스에 통합해 저장관리하는 시스템
```
---
> 데이터 베이스 시스템의 구성<br>
- 데이터 베이스 관리 시스템, 사용자, 처리 연산을 담당하는 컴퓨터
---
> 스키마<br>
- 데이터 베이스에 저장되는 데이터 구조와 제약조건을 정의
---
> 인터페이스<br>
- 스키마에 따라 데이터베이스에 실제로 저장된 값
---
> 3단계 데이터 베이스 구조<br>
```
1. 외부단계
- 사용자 관점
- 서브 스키마라고 불리기도함
- 사용자에게 필요한 데이터 베이스 정의
2. 개념단계
- 논리적 관점
- 구조와 제약조건등을 정의
3. 내부단계
- 물리적 장치와 관련
```
---
> 데이터 독립성<br>
- 하위 스키마를 변경하더라도 상위 스키마가 영향을 받지 않는 특성
- 데이터 베이스를 3단계로 나누고, 단계별로 스키마유지, 스키마사이의 대응관계를 정의하는 궁극적인 목적
---
> 최종사용자(end User)<br>
- 데이터 베이스에 접근하여 데이터를 조작하는 사람, 주로 데이터 조작어 이용
---
> 외부스키마와 개념스키마 대응관계<br>
- 응용 인터페이스
- 외부/개념 사상, 논리적 데이터 독립성
---
> 개념스키마와 내부스키마 대응관계<br>
- 저장 인터페이스
- 물리적 데이터 독립성
---
> 데이터 사전<br>
```
- 시스템 카탈로그
- 데이터 베이스에 저장되는 정보, 즉 메타데이터를 유지하는 시스템 데이터 베이스
- 스키마, 사상정보, 다양한 제약조건을 저장
- DBMS가 스스로 생성하고 유지함
- 일반사용자도 접근은 가능하고, 단 저장 내용 검색만 가능
```
---
> 데이터 언어<BR>
```
<사용목적에 따른 분류>
1. 데이터 정의어(DDL)
- 데이터 베이스 생성, 데이블삭제, 인덱스 생성등 => create, drop, alter등
2. 데이터 조작어(DML)
- 절차적 데이터 조작어 : 저급 데이터 단위, WHAT, HOW 모두 명세를 표기
- 비절차적 데이터 조작어 : 고급 데이터 단위, WHAT은 명세를 표기하지만, HOW는 시스템에 위임, 질의어 형태, 독자적& 대화식으로 동작
3. 데이터 제어어(DCL)
- 보안 및 데이터 정확성 유지
- 데이터 복구 및 병행제어의 기능 제공
```
---
> 데이터 모델링<br>
- 개녕적 모델링에서 논리적 모델링 순서로 변환
```
1. 개념적 모델링
- 사람의 머리로 이해할 수 있는 구조
2. 논리적 모델링
- 데이터 베이스에 적합한 형태로 변환한 데이터 모델
- 개체 -> 레코드
- 특성 -> 필드
```
---
> 데이터 모델 구성요소<br>
```
- 데이터 구조
- 연산
- 제약 조건
```
---
> 데이터 모델<br>
```
- 특정 DBMS는 특정 논리적 데이터 모델하나만 선정해 사용
- 단순 데이터 모델 -> 논리적 데이터 모델
- 개념적 데이터 모델 -> 속성들로 기술된 개체타입과 이 개체들의 타입들간의 관계를 이용해 현실세계를 표현
```
---
> E-R 다이어그램<br>
- 개체, 속성, 관계를 표현
- 개체관계의 개념적 모델링
```
(구성요소 및 표현)
- 개체 타입 : 직사각형
- 속성 : 타원형
- 유도 속성 : 점선 타원형
- 복합 속성 : 이중타원
- 관계 타입 : 다이아몬드
- 키 속성 : 밑줄 , 개체의 각 인스턴트들이 중복되지 안도록하는 중요한 역할, 즉 각각의 인스턴스를 구분할 수 있도록 함
- 필수적 참여 : 이중선, 개체 관계 모델에서 모든 개체인스턴스가 관계에 반드시 참여해야함을 의미
```
---
> 개체 타입<br>
- 개체가 가지고 있는 속성들의 집합
---
> 개체 인스턴스<br>
- 하나의 행의 형태로 개체타입에 입력된 데이터들의 집합
---
> 개체 집합<br>
- 개체 인스턴스들의 집합
---
> 오너개체와 약한 개체<br>
```
- 다른 존재여부를 결정하는 개체 -> 오너개체
- 오너 개체와 약한 개체는 일대다 관계
- 약한 개체는 오너 개체와 관계에 필수적으로 참여하는 특징이 있음
- 약한 개체는 오너 개체의 키를 포함하여 키를 구성하는 특징
```
