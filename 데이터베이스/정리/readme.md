> 데이터 베이스 시스템과 데이터 베이스 관리 시스템<br>
```
1. 데이터 베이스 시스템
- 데이터 베이스에 데이터를 저샂ㅇ하고, 이를 관리하여 조직에 필요한 정보를 생성해주는 시스템
- 데이터 베이스 관리시스템보다 더 큰 의미
2. 데이터 베이스 관리 시스템
- 조직에 필요한 데이터를 데이터 베이스에 통합해 저장관리하는 시스템
```
---
> 데이터 베이스 시스템의 구성<br>
- 데이터 베이스 관리 시스템, 사용자, 처리 연산을 담당하는 컴퓨터
---
> 스키마<br>
- 데이터 베이스에 저장되는 데이터 구조와 제약조건을 정의
---
> 인터페이스<br>
- 스키마에 따라 데이터베이스에 실제로 저장된 값
---
> 3단계 데이터 베이스 구조<br>
```
1. 외부단계
- 사용자 관점
- 서브 스키마라고 불리기도함
- 사용자에게 필요한 데이터 베이스 정의
2. 개념단계
- 논리적 관점
- 구조와 제약조건등을 정의
3. 내부단계
- 물리적 장치와 관련
```
---
> 데이터 독립성<br>
- 하위 스키마를 변경하더라도 상위 스키마가 영향을 받지 않는 특성
- 데이터 베이스를 3단계로 나누고, 단계별로 스키마유지, 스키마사이의 대응관계를 정의하는 궁극적인 목적
---
> 최종사용자(end User)<br>
- 데이터 베이스에 접근하여 데이터를 조작하는 사람, 주로 데이터 조작어 이용
---
> 외부스키마와 개념스키마 대응관계<br>
- 응용 인터페이스
- 외부/개념 사상, 논리적 데이터 독립성
---
> 개념스키마와 내부스키마 대응관계<br>
- 저장 인터페이스
- 물리적 데이터 독립성
---
> 데이터 사전<br>
```
- 시스템 카탈로그
- 데이터 베이스에 저장되는 정보, 즉 메타데이터를 유지하는 시스템 데이터 베이스
- 스키마, 사상정보, 다양한 제약조건을 저장
- DBMS가 스스로 생성하고 유지함
- 일반사용자도 접근은 가능하고, 단 저장 내용 검색만 가능
```
---
> 데이터 언어<BR>
```
<사용목적에 따른 분류>
1. 데이터 정의어(DDL)
- 데이터 베이스 생성, 데이블삭제, 인덱스 생성등 => create, drop, alter등
2. 데이터 조작어(DML)
- 절차적 데이터 조작어 : 저급 데이터 단위, WHAT, HOW 모두 명세를 표기
- 비절차적 데이터 조작어 : 고급 데이터 단위, WHAT은 명세를 표기하지만, HOW는 시스템에 위임, 질의어 형태, 독자적& 대화식으로 동작
3. 데이터 제어어(DCL)
- 보안 및 데이터 정확성 유지
- 데이터 복구 및 병행제어의 기능 제공
```
---
> 데이터 모델링<br>
- 개녕적 모델링에서 논리적 모델링 순서로 변환
```
1. 개념적 모델링
- 사람의 머리로 이해할 수 있는 구조
2. 논리적 모델링
- 데이터 베이스에 적합한 형태로 변환한 데이터 모델
- 개체 -> 레코드
- 특성 -> 필드
```
---
> 데이터 모델 구성요소<br>
```
- 데이터 구조
- 연산
- 제약 조건
```
---
> 데이터 모델<br>
```
- 특정 DBMS는 특정 논리적 데이터 모델하나만 선정해 사용
- 단순 데이터 모델 -> 논리적 데이터 모델
- 개념적 데이터 모델 -> 속성들로 기술된 개체타입과 이 개체들의 타입들간의 관계를 이용해 현실세계를 표현
```
---
> E-R 다이어그램<br>
- 개체, 속성, 관계를 표현
- 개체관계의 개념적 모델링
```
(구성요소 및 표현)
- 개체 타입 : 직사각형
- 속성 : 타원형
- 유도 속성 : 점선 타원형
- 복합 속성 : 이중타원
- 관계 타입 : 다이아몬드
- 키 속성 : 밑줄 , 개체의 각 인스턴트들이 중복되지 안도록하는 중요한 역할, 즉 각각의 인스턴스를 구분할 수 있도록 함
- 필수적 참여 : 이중선, 개체 관계 모델에서 모든 개체인스턴스가 관계에 반드시 참여해야함을 의미
```
---
> 개체 타입<br>
- 개체가 가지고 있는 속성들의 집합
---
> 개체 인스턴스<br>
- 하나의 행의 형태로 개체타입에 입력된 데이터들의 집합
---
> 개체 집합<br>
- 개체 인스턴스들의 집합
---
> 오너개체와 약한 개체<br>
```
- 다른 존재여부를 결정하는 개체 -> 오너개체
- 오너 개체와 약한 개체는 일대다 관계
- 약한 개체는 오너 개체와 관계에 필수적으로 참여하는 특징이 있음
- 약한 개체는 오너 개체의 키를 포함하여 키를 구성하는 특징
```
---
> 개체-관계 모델<br>
```
- 현실 세계를 사람들의 머릿속에 그릴 수 있는 개념적으로 구조를 모델링 => 개념적 데이터 모델
- 어떤 DBMS로 DB를 구축하든 상관없음
```
---
> 논리적 데이터 모델링<br>
- DBMS의 종류가 중요
---
> 계층 데이터 모델<br>
- 특징
```
- DB의 논리적 구조가 tree형태
- 개체간의 상하관계가 성립
- 두 개체 사이에 하나의 관계만 정의 가능
- 개념적 구조를 모델링하기 hard -> 구조가 복잡해질 수 있음
```
- 장점
```
- 구조가 간단
- 판독이 용이
```
- 단점
```
- 데이터 상호간 유연성 낮음
- 제약이 많음 -> 구조가 복잡해질 수 있음
- 데이터 삽입, 삭제, 수정등의 연산이나 데이터 검색이 어려움
```
---
> 네트워크 데이터 모델<br>
```
- DB의 논리적 구조가 그래프형태
- 일대다 관계만 허용
- 오너 & 멤버 관계
- 두 개체사이에 여러관계를 정의 가능 -> 이름으로 구별
- 구조 복잡, 데이터의 삽입, 삭제, 수정, 검색이 쉽지 않음
```
---
> 관계 데이터 모델<br>
```
- 개념적 구조 -> 논리적 구조로 표현
- 컴퓨터 환경에 저장하기보다 쉬운 형태로 바꾸는 역할
- 개체를 릴레이션(테이블)형태로 표현
- 열은 개체의 특징을 나타내는 속성 혹은 애브리뷰트의 집합으로 구성
- 각 속성들의 값을 모다 그 개체가 표현하는 실제 대상의 데이터들을 하나의 튜플이라고 한다
```
- 속성의 수 : degree, 차수
- 저장되어 있는 데이터들의 행의 수 : 기수, 카디널리티
---
> 릴레이션의 특성<br>
```
- 튜플의 무순서
- 튜플의 유일성
- 속성의 무순서
- 속성의 원자성
```
---
> 릴레이션 스키마<br>
- 릴레이션을 구성하는 속성 -> 애브리뷰트의 집합
---
> 관계 데이터 모델의 키<br>
- 특성
```
- 유일성 : 하나의 릴레이션에서 모든 튜플은 서로 다른 키 값을 가져야함을 의미
- 최소성 : 꼭 필요한 최소한의 속성들로만 키를 구성한다는 의미
```
- 종류
```
- 슈퍼키 : 유일성을 만족하는 키의 집합
- 후보키 : 유일성과 최소성을 만족하는 속성의 부분집합
- 기본키 : 후보키를 중 선정한 하나의 속성
- 대체키 : 기본키를 제외한 나머지 키
- 외래키 : 다른 릴레이션의 기본키를 참조하는 속성 또는 속성들의 집합
          릴레이션들 간의 관계를 표현
          참조하는 릴레이션 : 외래키를 가진 릴레이션
          속성에는 null값 사용가능
```
---
> 관계 데이터 모델의 제약조건<br>
```
- 개체 무결성 : 기본키는 null값을 가질 수 없음을 의미
- 참조 무결성 : 외래키는 참조할 수 없는 값을 가질 수 없는 규칙(null값이 아닌 없는 값을 가질 수 없음을 의미)
```
---
> 데이터 구조<br>
- 데이터를 저장하고 있는 구조와 방법을 의미
---
> 제약조건<br>
- 데이터 구조와 연산이 지켜야 할 규칙
---
> 연산<br>
- 저장되어 있는 데이터를 조회 or 처리하는 방법들에 대한 내용을 정의
---
> 관계형 데이터 모델의 연산<br>
```
- 릴레이션의 정의, 데이터 조작, 데이터 베이스 시스템의 제어들에 사용됨
- 관계 대수와 관계해석이 있음 -> 데이터 처리를 위해 명시하는 내용만 다르고, 기능과 표현적 측면에서 성능의 차이는 존재X
```
---
> 관계 대수<br>
```
- 절차적 언어
- 원하는 결과를 얻기위해 데이터 처리 과정을 순서대로 기술
(연산자의 종류)
1. 일반 집합 연산자
    - 합집합
    - 교집합
    - 차집합
    - 카티션 프로젝트 : 합병이 가능, 값이 존재하지 않아도 연산이 가능
          * 합병 : 두 릴레이션의 차수가 같고, 서로 대응되는 속성의 도메인이 같음을 의미
2. 순수 관계 연산자
    - 조인 : 공통속성을 이용, 투플들을 연결하여 만들어진 투플들을 반환 ⋈
    - 프로젝트 : 특정속성을 검색
    - 디비전 : 한 릴레이션의 모든 투플과 관련있는 다른 한 릴레이션의 투플로 결과 릴레이션 구성
    - 셀렉트 : 조건을 만족하는 투플들을 검색해 반환
```

```
- 외부 조인 : 연상에서 제외되는 모든 투플을 결과 릴레이션에 포함
```
