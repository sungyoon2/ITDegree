# 토론
---
> 주제<br>
- 스레드의 상태변화의 특징에 대해 토론하시오
> 제출<br>
```
- 스레드는 프로세스에서 실행 제어만 분리한 실행단위로 이를 사용하면서 프로세스안에서 공통의 목적 달성을 위해 병렬로 수행합니다.
- 스레드의 상태 변화는 프로그램의 동시성과 효율성을 이해하는 데 매우 중요한 개념입니다.
- 스레드는 항상 스레드하나만 실행되며, 아래의 과정을 복합적으로 거치며 수행이 됩니다.
- 스레드는 일반적으로 New(생성), Runnable(준비), Running(실행), Blocked/Waiting(대기), Terminated(종료) 상태를 거칩니다.
- 스레드가 생성되면 new 상태로 시작하고, 실행 준비가 되면 Runnable 상태로 이동합니다.
- 이후 CPU에 의해 실행되면 Running상태가 됩니다.
- 특히 Blocked/Waiting 상태는 리소스가 부족하거나 다른 스레드의 작업이 끝나기를 기다릴 때 발생하며, 또한 이 과정에서 다른 스레드를 사용가능한 상태이기에 효율적인 리소스 관리가 중요합니다.
- 마지막으로 작업이 완료되면 Terminated상태가 됩니다.
- 스레드 상태변화는 주로 운영체제의 스케쥴러와 자원 할당 정책에 의해 결정되며, 이를 잘 이해하면 데드락이나 성능저하 같은 문제를 예방할 수 있습니다
- 상태 전환이 적을수록 스레드 관리가 효율적이라는 점도 기억해야합니다.
```
